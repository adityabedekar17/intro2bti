# About the author

**Aditya Ashish Bedekar** is a PhD student at University of California Santa Cruz in the [Hardware Systems Collective](https://hsc.ucsc.edu/), advised by [Dr. Dustin Richmond](https://www.dustinrichmond.com/). His research focuses on physics-based degradation modeling for advanced process nodes, and research interests lie in hardware reliability and hardware security.

This resource was built to give hardware security and semiconductor reliability researchers a single, accessible entry point to BTI, which did not exist, and would immensly help the research community. 

{: .note }
If others from the community contributed corrections or paper suggestions, they will be acknowledged here by name with permission. If you submitted feedback and would like to be credited, please [Email me](mailto:abedekar@ucsc.edu).

## Cite this resource

If this resource was useful in your research or teaching, please cite it. All four formats below are equivalent, please do use whichever is the most relevant.


### BibTeX

```bibtex
@misc{A.A. Bedekar 2026,
  author       = {Bedekar, Aditya Ashish},
  title        = {Introduction to {BTI}  A resourcefor Hardware Security
                  and Semiconductor Reliability Researchers},
  year         = {2026},
  howpublished = {\url{https://adityabedekar17.github.io/intro2bti}},
  note         = {Accessed: \today}
}
```
### Plain text

```
A.A. Bedekar. " Introduction to BTI." 2026.
https://adityabedekar17.github.io/intro2bti
```



## Use in courses

You are welcome to link to or share this resource in course materials, syllabi, or reading lists without asking for permission. A heads-up email is appreciated but not required.

If you are a course instructor and find this useful for a graduate hardware security or computer architecture course, [let me know](mailto:abedekar@ucsc.edu) I am happy to discuss additions or adjustments for a specific course context.
